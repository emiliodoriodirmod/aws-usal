AWS opera en 33 regiones geográficas con 105 zonas de disponibilidad alrededor del mundo

Regiones AWS

Una región es una ubicación geográfica física donde agrupamos los centros de datos. Cada región de AWS está completamente aislada y consta de múltiples zonas de disponibilidad.

Ejemplos de Regiones

Singapur

Región Asia Pacífico

Consideraciones al elegir una región

  • Cumplimiento y soberanía de datos: Requisitos legales y regulatorios
  • Proximidad a los usuarios: Reducir latencia
  • Disponibilidad de servicios: No todos los servicios están en todas las regiones
  • Precios: Los costos varían entre regiones

Zonas de Disponibilidad

Las zonas de disponibilidad (AZ) son centros de datos aislados dentro de una región de AWS.

Características clave

Alta disponibilidad

Diseñe aplicaciones que se ejecuten en múltiples AZ para lograr tolerancia a fallas.

Baja latencia

Las AZ están conectadas con redes privadas de alto ancho de banda.

Aislamiento de fallas

Los problemas en una AZ no afectan a las demás.

Servicios de Cómputo de AWS

EC2

Amazon Elastic Compute Cloud

Capacidad de cómputo redimensionable en la nube.

  • Máquinas virtuales (instancias)
  • Control completo del sistema operativo
  • Múltiples tipos de instancia optimizados
  • Pago por segundo u hora de uso
ECS

Amazon Elastic Container Service

Servicio de orquestación de contenedores altamente escalable.

  • Ejecuta aplicaciones en contenedores Docker
  • Integración con AWS Fargate para serverless
  • Service discovery y load balancing integrados
  • Compatible con Amazon ECR
SERVERLESS

AWS Lambda

Ejecute código sin pensar en servidores.

  • Ejecuta código en respuesta a eventos
  • Escala automáticamente
  • Pago por milisegundo de ejecución
  • Soporta múltiples lenguajes de programación

Comparación de Servicios de Cómputo

Característica Amazon EC2 Amazon ECS AWS Lambda
Modelo Máquinas virtuales Contenedores Funciones
Tiempo de inicio Minutos Segundos Milisegundos
Escalabilidad Manual/Auto Scaling Automática Instantánea
Precio Por hora/segundo Por recurso utilizado Por invocación y duración
Gestión Usuario gestiona OS AWS gestiona contenedores Sin servidor
Casos de uso Aplicaciones tradicionales Microservicios Event-driven, APIs
Tiempo máximo Sin límite Sin límite 15 minutos
Persistencia Con EBS volumes Con EFS/EBS Sin estado
Recomendación: Elija EC2 para control total, ECS para aplicaciones containerizadas, y Lambda para cargas de trabajo event-driven y de corta duración.

Código de Demostración

Ejecutaremos este código Python en los tres servicios de cómputo:

#!/usr/bin/env python3
import json
import platform
import socket
from datetime import datetime

def process_data(event=None):
    """
    Función que procesa datos y retorna información del sistema
    Compatible con EC2, ECS y Lambda
    """
    
    # Obtener información del sistema
    current_time = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    
    # Construir respuesta
    data = {
        "timestamp": current_time,
        "service": "AWS Compute Demo",
        "hostname": socket.gethostname(),
        "platform": platform.platform(),
        "python_version": platform.python_version(),
        "status": "success",
        "message": "¡Hola desde AWS!"
    }
    
    # Si hay evento (Lambda), incluirlo
    if event:
        data["event_received"] = event
    
    return {
        "statusCode": 200,
        "body": json.dumps(data, indent=2)
    }

# Entry point para EC2/ECS
if __name__ == "__main__":
    result = process_data()
    print(result['body'])

# Handler para Lambda
def lambda_handler(event, context):
    return process_data(event)

Ejemplo de uso con Python

import boto3
import json

# Cliente de Bedrock
bedrock = boto3.client(
    service_name='bedrock-runtime',
    region_name='us-east-1'
)

# Preparar prompt
prompt = "Explica los beneficios de usar AWS para machine learning"

# Invocar modelo
response = bedrock.invoke_model(
    modelId='anthropic.claude-v2',
    contentType='application/json',
    accept='application/json',
    body=json.dumps({
        "prompt": f"\n\nHuman: {prompt}\n\nAssistant:",
        "max_tokens_to_sample": 300,
        "temperature": 0.7,
        "top_p": 1,
    })
)

# Procesar respuesta
result = json.loads(response['body'].read())
print(result['completion'])
